## 规则查询方法

### 按规则号查找
```bash
# 规则号格式: XXX.Y (如 702.19 践踏)
grep -n "<b id='cr702-19'" markdown/7.md
```

### 按中文关键词查找
```bash
# 中文词汇表（拼音排序）
grep -B2 -A3 "践踏" markdown/glossarycn.md

# 或直接搜索关键字
grep -n "span id=cr702" markdown/7.md | grep 践踏
```

## 牌面信息查询（涉及具体牌张时）

### 1. 获取官方中文牌名
```bash
curl -s "https://mtgch.com/api/v1/autocomplete/?q={牌名}&is_for_deck=false&size=1&page=1" \
  -H "accept: application/json" | head -c 5000
```
提取字段：`display_name`（中文牌名）、`atomic_translated_type`（中文类别）

### 2. 获取完整异能文本
```bash
curl -s "https://api.scryfall.com/cards/named?fuzzy={牌名}" 2>/dev/null | head -c 5000
```
提取字段：`oracle_text`（英文异能）、`type_line`、`mana_cost`

## 规则分析框架

### 关键原则：查证牌张异能

**⚠️ 切勿凭记忆回答涉及具体牌张的问题。** 即使你认为熟悉某张牌，也必须先查证其完整异能文本。

常见陷阱：
- Roiling Vortex 触发条件是"未花费法术力"而非"从非手牌区域"
- Suspend 只能从手牌延迟，不能从其他区域

**每次涉及具体牌张时，执行：**
```bash
curl -s "https://api.scryfall.com/cards/named?fuzzy={牌名}" | grep -o '"oracle_text":"[^"]*"'
```

### 框架1：层系统判定（613）

**层顺序：**
```
层1: 复制
层2: 操控权
层3: 文字栏
层4: 类别 ← 腥红之月、乌尔博格在此
层5: 颜色
层6: 异能 ← 潮缚师移除异能在此
层7: 力量/防御力
```

**关键判断：**

| 情况 | 规则 | 应用 |
|------|------|------|
| 效应在**不同层** | 613.6 跨层效应 | 各层独立生效，即使源异能消失 |
| 效应在**同一层** | 613.8 从属关系 | 判断哪个效应"依赖"另一个 |

**从属关系判断：** 若应用A会改变B的存在或结果，则B从属于A，A先应用。

**时间印记问题（613.7）- 复杂场景：**

当效应涉及"创造永久物并获得异能"时：
1. 创造永久物是事件
2. "获得异能"是单独的持续性效应，不是印刷文字
3. 需要比较"获得异能"与其他效应的时间印记

**案例：Sokenzan 创造衍生物 vs Dress Down**
- Dress Down 先结算 → 时间印记早
- 衍生物后创造 → "获得敏捷"时间印记晚
- 最终：先移除所有异能，然后获得敏捷 → 衍生物有敏捷

参考规则：113.1a, 113.10, 608.2c, 613.7

### 框架2：堆叠与区域判定

**核心定义：**
- **112.1**: 堆叠中的牌都是"咒语"
- **110.1**: 只有战场上的牌是"永久物"
- **112.2**: 每个咒语都有操控者

**判断流程：**
1. 效应提到"你操控的永久物" → 仅战场上的牌
2. 效应提到"你操控的咒语" → 仅堆叠上的牌
3. "施放时触发"的异能结算时 → 咒语可能仍在堆叠，未成为永久物

### 框架3：异能类型判定

**启动式异能** vs **触发式异能** vs **静止式异能**

| 类型 | 标识 | 结算时机 |
|------|------|----------|
| 启动式 | "[费用]:[效应]" | 玩家主动启动，进入堆叠 |
| 触发式 | "当/每当/在[事件]时" | 事件发生时自动触发，APNAP顺序入堆叠(603.3) |
| 静止式 | 持续生效的描述 | 不进入堆叠，立即生效或持续生效 |

**多个触发式异能同时触发：** APNAP顺序（101.4）→ 主动牌手先放，非主动牌手后放 → 后放的先结算

### 框架4：回合结构与优先权（500/504）

**关键步骤的优先权窗口：**

| 步骤 | 优先权传递时机 | 常见误区 |
|------|---------------|----------|
| 抽牌步骤 | 抽牌**后**获得优先权 (504.2) | 误以为抽牌后直接进入主阶段 |
| 主阶段 | 回合开始时、动作后 | 法术牌只能在此阶段施放 |
| 战斗开始步骤 | 双方获得优先权 | - |
| 战斗伤害步骤 | 伤害结算后 | - |

**判断流程：**
1. 确定当前步骤
2. 检查该步骤是否有优先权窗口
3. 主动牌手先获得优先权，然后非主动牌手

### 框架5：延迟（Suspend）规则（702.62）

**核心限制：牌只能从手牌延迟** (702.62a)

- 从放逐区"施放"牌 ≠ 延迟
- Ragavan 等牌允许从放逐区施放，但不能延迟
- 延迟是特殊的起动式异能，只能在手牌中起动

### 框架6：替代式效应与防止式效应（614/615）

**替代式效应特征：** "改为..."、「如果...则改为...」
- 614.8: 替代效应完全替代原事件

**防止式效应特征：** 「防止...」
- 615.5: 防止伤害的同时可触发其他效应

### 框架7：多部分咒语的结算（608.2c）

多模式咒语或包含多个效果的咒语：**按叙述顺序依次结算**，不是同时发生。

## 回答流程

1. **理解问题** → 明确情境和涉及的牌/效应
2. **查询牌面** → 如有不明牌张，调用API查询
3. **定位规则** → 使用Grep查找相关规则条文
4. **应用框架** → 选择适用的分析框架（层系统/堆叠/异能类型）
5. **逐步推演** → 按规则逻辑得出结果
6. **给出结论** → 清晰回答，附规则依据

## 常见术语辨析

| 错误表述 | 正确理解 |
|----------|----------|
| "堆叠上的永久物" | 堆叠上的"永久物咒语"，结算后才成为永久物 |
| "先攻造成双倍伤害" | 先攻伤害步骤先造成伤害，普通生物尚未造成 |
| "红月vs乌尔博格看时间印记" | 层4从属关系，乌尔博格从属于红月 |
| "重生在受到致命伤害时触发" | 重生只在"将要被消灭"时触发，伤害本身不触发 |
| "Roiling Vortex 是非手牌区域触发" | Roiling Vortex 触发条件是"未花费法术力" |
| "可以从放逐区延迟牌" | Suspend 只能从手牌延迟 (702.62a) |
| "抽牌后直接进入主阶段" | 抽牌步骤抽牌后有优先权窗口 (504.2) |
| "创造衍生物并获得异能是原子的" | "获得异能"是单独效应，需应用时间印记 |
